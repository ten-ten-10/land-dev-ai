# TomoAid — 模組化 GTM 導航手冊
# Go-to-Market Master Playbook

> **版本：** v1.1
> **建立日期：** 2026-04-15
> **適用範圍：** TomoAid 旗下所有產品線（TomoLand、SaaaS、TomoHire）

---

## 前言：為什麼需要通用 GTM Framework？

TomoAid 產品跨度大（土地開發 AI、技術引擎服務、人才招募系統），若針對每個產品從頭設計打法，將耗費大量資源且難以管理。本手冊的目標不是統一「動作」，而是統一**思考框架**與**決策流程**，讓不同產品的負責人都能快速、有邏輯地推向市場。

---

## Phase 0｜品牌建設（Branding）— GTM 的靈魂與地基

> GTM 決定「怎麼賣」，Branding 決定「為什麼信任你」。沒有品牌地基就衝 GTM，等同於開了一間裝潢精美的店，但路人看招牌不知道你是誰。

### Branding vs. GTM 的分工

| 維度 | Branding（品牌） | GTM（市場進入） |
|------|-----------------|----------------|
| **核心問題** | 我們是誰？我們為什麼存在？ | 我們要賣給誰？怎麼賣？ |
| **目標** | 建立信任感與心理位子（Mindshare） | 獲取市佔率與營收（Market Share） |
| **時效** | 長期、持續、累積 | 短期、爆發、具備明確時程 |
| **比喻** | 地基與人設（這個人穩不穩？幽默嗎？） | 約會計畫（去哪吃飯？穿什麼衣服？） |

### 為什麼 TomoAid 更需要先做 Branding？

TomoAid 產品跨度大，若沒有統一品牌形象：
- 每個產品上市都像「孤兒獨立作戰」，無法互相背書
- 行銷資源每次從零開始，CAC 居高不下
- 難以建立溢價能力，只能打價格戰

**目標：建立「保護傘」效應** — 讓 TomoLand、SaaaS、TomoHire 在市場上看起來都像「同一個家族」的作品。

### 品牌曝光渠道規劃

#### 1. 知識型自媒體（優先推薦）

> 核心主軸：**「TomoAid 是幫助企業實現 AI 自動化改革的夥伴」**
> 內容目的不是教工具，而是持續堆疊「TomoAid 很懂 AI 自動化」的品牌印象。

**內容框架原則：**
```
❌ 「Claude 怎麼建立 Skill」（工具導向）

✅ 「我們如何把一個 3 天的招募初篩流程壓縮到 2 小時
    —— 以及背後用了什麼 AI 工具」（結果導向，工具是附帶說明）
```

**五大內容分類：**

| 分類 | 核心問題 | 具體內容方向 | 比例 |
|------|---------|------------|------|
| **AI 轉型思維** | 企業如何開始 AI 化？ | 找到第一個自動化切入點、常見迷思破解 | 25% |
| **工作流改造案例** | 真實的流程長什麼樣？ | 「原本 3 小時，現在 15 分鐘」的實例展示 | 30% |
| **工具實戰教學** | 怎麼做到的？ | AI 工具（Claude、n8n 等）作為方法示範 | 20% |
| **產業 AI 應用** | 我的行業怎麼用？ | HR、建商、新創各自的 AI 自動化場景 | 15% |
| **品牌 Why** | TomoAid 是誰？ | 使命、創辦人觀點、我們看到的未來 | 10% |

**平台策略：**

| 平台 | 內容方向 | 適合觸及 | 優先度 |
|------|----------|---------|--------|
| **LinkedIn** | 工作流改造案例、AI 轉型思維、創辦人觀點 | 企業決策者、HR、地產業者 | ⭐⭐⭐ 最高 |
| **官網 Blog** | 深度教學、產業 AI 應用案例（SEO） | 長尾搜尋流量、建立專業形象 | ⭐⭐⭐ 最高 |
| **X / Threads** | 短觀點、工具小技巧、流程改造片段 | 科技圈、創作者 | ⭐⭐ 中 |
| **YouTube / IG Reels** | 工作流改造示範影片、產品 Demo | 廣泛受眾、視覺型用戶 | ⭐⭐ 中 |

**內容範例（結果導向）：**
- 「我們如何用 AI 把土地評估週期從 3 週壓到 3 天」（TomoLand）
- 「一個 HR 團隊如何用 AI 把初篩時間縮短 80%」（TomoHire）
- 「為什麼大部分企業的 AI 導入都失敗——以及怎麼避免」（品牌觀點）

#### 4. AI 內容自動化工作流

> 將 GTM 策略轉化為可持續產出的內容引擎，降低人工成本並維持發布頻率。

**工作流架構：**

```
GTM 主題庫（JSON）
  ↓
Claude API 內容生成
（平台 + 受眾 + 主題 → 草稿）
  ↓
人工審核（GitHub Issue：tag content-draft）
  ↓
排程發布工具（Buffer / n8n）
  ↓
LinkedIn / Blog / Threads
```

**三個建置層次：**

| 層次 | 內容 | 時程 |
|------|------|------|
| **Level 1（半自動）** | 整理 GTM 主題庫 + Prompt Template，Claude API 產出草稿，人工發布 | Q2 Goal 4.1–4.2 |
| **Level 2（定期自動生成）** | 每週自動觸發生成草稿，存入 GitHub Issue 待審核後發布 | Q2 Goal 4.3 |
| **Level 3（全自動，進階）** | 審核通過後自動排程發布至各平台 | Q2 後規劃 |

**GTM 主題庫對應：**

| 產品 | 內容方向 | 目標受眾 |
|------|---------|---------|
| TomoLand | 「建商如何用 AI 縮短土地評估週期」系列 | 建商、地產業者 |
| TomoHire | 「HR 用 AI 招募」實戰技巧系列 | HR 主管 |
| SaaaS | 「RAG / Embedding 是什麼」技術科普系列 | 開發者、技術決策者 |
| TomoAid 母品牌 | AI co-work 觀點、創辦人視角 | 廣泛 B2B 受眾 |

---

#### 5. 執行優先順序

| 順序 | 行動 | 原因 |
|------|------|------|
| **1** | 官網補產品頁 | 成本最低，讓客戶「查得到你」 |
| **2** | 整理 GTM 主題庫 + 建立 AI 內容腳本 | 讓後續內容產出有系統、可持續 |
| **3** | LinkedIn 開始發內容（透過 AI 系統） | 直接觸及 B2B 決策者 |
| **4** | 官網 Blog 建立 | 長期 SEO 流量累積 |
| **5** | 尋找 1–2 個合作夥伴 | 快速借力擴散，降低冷啟動成本 |

## 第三階段｜核心策略框架 — 五問法 (The 5-Question Framework)

> 每個產品在啟動 GTM 前，負責人必須回答以下五個問題。

---

### 🏢 TomoLand — AI 土地開發平台

| 問題 | 答案 |
|------|------|
| **1. 解決誰的什麼痛苦？** | 土地開發商、建商在找地、評估、買地流程中，需耗費大量人力手動蒐集資訊、試算，且容易遺漏關鍵風險 |
| **2. 替代方案是什麼？** | 自行聘請調查人員、委託仲介、手動試算 Excel、人脈網絡 |
| **3. 核心優勢（USP）？** | AI 整合公開情報（法拍、公報）+ 自動 ROI 試算 + 土地背景調查，大幅縮短評估週期 |
| **4. 購買路徑？** | 行業展覽認識 → 業務展示 Demo → 顧問提案 → 簽約 |
| **5. 成功指標（3個月後）？** | Demo 預約數、簽約數、每案評估時間縮短 % |

---

### ⚙️ SaaaS — AI 技術引擎平台

| 問題 | 答案 |
|------|------|
| **1. 解決誰的什麼痛苦？** | 中小型 AI 應用開發商難以自建穩定的 Scraping、Embedding、RAG 能力，開發成本高且不穩定 |
| **2. 替代方案是什麼？** | 自建工程團隊、使用 Apify / Diffbot 等國際工具、或放棄該技術方向 |
| **3. 核心優勢（USP）？** | 實戰驗證的技術模組（TomoLand 自用）、中文語料優化、快速整合 API |
| **4. 購買路徑？** | GitHub / 技術社群曝光 → 官網文件自助試用 → 升級付費方案 |
| **5. 成功指標（3個月後）？** | API 試用數、付費轉換率、MRR |

---

### 👥 TomoHire — AI 驅動招募平台

| 問題 | 答案 |
|------|------|
| **1. 解決誰的什麼痛苦？** | HR 團隊每天花大量時間篩選不符合的履歷，且主動尋源效率低 |
| **2. 替代方案是什麼？** | 手動篩選 104 / 人力銀行、委外獵頭、傳統 ATS 系統 |
| **3. 核心優勢（USP）？** | 自動爬取 104 VIP 履歷 + AI 評分契合度 + 個人化面試題生成，一站式全流程 |
| **4. 購買路徑？** | HR 社群 / LinkedIn 廣告 → 官網試用申請 → Inside Sales 跟進 → 簽約 |
| **5. 成功指標（3個月後）？** | 試用申請數、HR Demo 轉換率、每招募案縮短天數 |
